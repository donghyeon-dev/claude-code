Architecture Overview

Claude Code CLI는 Anthropic이 개발한 공식 AI 코딩 어시스턴트로, 터미널에서 직접 실행되는 대화형 도구입니다. TypeScript/React(Ink) 기반으로 구축되었으며, Registry-Driven Architecture 패턴을 따릅니다.

⚙ Registry-Driven Architecture
Commands, Tools, Skills, Plugins, Hooks가 모두 디렉토리 기반 레지스트리에서 조립됩니다. 런타임에 필터링/병합/중복제거가 수행됩니다.
🔒 Fail-Closed Security
모든 쓰기/실행 도구는 명시적 승인 필요 (기본 거부). 읽기 도구만 기본 허용입니다. ML 기반 자동 승인도 지원합니다.
🚀 Lazy Loading
React, SDK, OpenTelemetry 등 무거운 모듈은 지연 로딩됩니다. Feature Flag로 미사용 코드는 빌드시 제거(DCE)됩니다.
👥 Multi-Agent System
Leader-Worker 패턴의 Swarm 시스템으로 여러 에이전트가 병렬 작업합니다. UDS 소켓/메일박스 패턴으로 통신합니다.

Tools (40+)

Claude Code가 사용하는 모든 도구입니다. 각 도구는 tools/{ToolName}/ 디렉토리에 정의되며, buildTool() 패턴으로 구축됩니다.

각 도구 디렉토리에는 메인 구현({ToolName}.tsx), 시스템 프롬프트(prompt.ts), UI 컴포넌트(UI.tsx)가 포함됩니다.

RequiresPermission
N NotebookEdit
Jupyter 노트북 셀을 편집합니다. insert, delete, replace 작업을 지원합니다.

Tool Interface Contract

interface Tool {
  name: string
  description: string
  disabled?: boolean
  isEnabled?(): boolean

  schema: z.ZodType           // Zod 스키마로 입력 검증
  prompt?: string              // 시스템 프롬프트에 주입될 사용 지침
  defer?: boolean              // 메인 쿼리 루프 외부에서 실행
  readOnly?: boolean            // 상태를 변경하지 않는 도구
  concurrency?: number          // 최대 병렬 실행 수

  execute(input, context): Promise<ToolResult>
}

Permission System

Claude Code는 Fail-Closed 보안 모델을 사용합니다. 모든 쓰기/실행 작업은 명시적 승인이 필요하며, 읽기 작업만 기본 허용됩니다.

Permission Modes

D default
표준 모드. 각 도구 사용시 사용자에게 개별 승인을 요청합니다. 가장 안전한 모드입니다.
A auto
ML 기반 자동 승인 모드. Classifier가 위험도를 평가하여 자동 승인/거부합니다. 2단계 평가 (fast + thinking).
E acceptEdits
파일 편집을 자동으로 수락합니다. Bash 등 실행 도구는 여전히 승인이 필요합니다.
B bypassPermissions
모든 도구가 사전 승인됩니다. 위험! 개발/테스트 환경에서만 사용하세요.
P plan
계획 모드. 제한된 도구만 사용 가능하며, 실행 없이 미리보기만 합니다.
N dontAsk
사전 승인되지 않은 작업은 자동 거부합니다. CI/CD 환경에 적합합니다.

Permission Behaviors

allow - 허용 deny - 거부 ask - 사용자에게 질문

Permission Rules

규칙은 여러 소스에서 정의될 수 있으며, 우선순위가 있습니다:

1 policySettings — 조직/MDM 정책 (최우선)
↓
2 session — 현재 세션에서 설정된 규칙
↓
3 cliArg — CLI 인수로 전달된 규칙
↓
4 flagSettings — --settings 플래그
↓
5 localSettings — .claude/settings.local.json
↓
6 projectSettings — .claude/settings.json
↓
7 userSettings — ~/.claude/settings.json

Rule Patterns

// 특정 도구 허용
"Bash(git:*)"              // 모든 git 명령 허용
"Bash(npm test:*)"         // npm test 명령 허용
"Edit"                      // Edit 도구 전체 허용

// 특정 패턴 거부
"Bash(rm:-rf /*)"          // rm -rf / 거부
"Bash(curl:*)"             // curl 명령 거부

Permission Decision Flow

Allow Decision: 도구 실행 허용. 선택적으로 updatedInput으로 입력 수정 가능.
Ask Decision: 사용자에게 승인 요청. 메시지와 권한 업데이트 제안 포함.
Deny Decision: 실행 거부. 거부 사유를 포함합니다.

MCP (Model Context Protocol)

MCP는 Claude Code가 외부 도구 서버와 통신하는 프로토콜입니다. 7가지 전송 방식을 지원하며, 도구, 리소스, 프롬프트를 동적으로 로드합니다.

Supported Transports

▶ stdio
표준 입출력 서브프로세스 전송. 가장 일반적인 방식입니다. 로컬 MCP 서버 실행에 사용됩니다.
⇵ sse
Server-Sent Events (HTTP 스트리밍). 원격 서버와의 실시간 통신에 사용됩니다.
⇄ http
HTTP 요청/응답. REST API 스타일의 MCP 서버에 사용됩니다.
⇋ ws
WebSocket. 양방향 실시간 통신이 필요한 서버에 사용됩니다.
IDE sse-ide
IDE 확장용 SSE. VS Code 등 IDE 확장과의 내부 통신에 사용됩니다.
SDK sdk
SDK 인프로세스 (JavaScript/TypeScript). 같은 프로세스 내에서 실행됩니다.
P claudeai-proxy
Claude.ai 프록시 서버. 출력 전용 모드로 작동합니다.

Configuration Examples

// stdio - 로컬 서브프로세스
{
  "type": "stdio",
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem"],
  "env": { "KEY": "value" }
}

// HTTP/SSE - 원격 서버
{
  "type": "http",
  "url": "https://mcp-server.example.com",
  "headers": { "Authorization": "Bearer token" },
  "oauth": { "clientId": "..." }
}

// WebSocket
{
  "type": "ws",
  "url": "ws://localhost:8080"
}

Agent / Subagent System

Claude Code는 다중 에이전트 시스템을 지원합니다. 메인 에이전트가 서브에이전트를 생성하여 병렬 작업을 수행할 수 있습니다.

Task Types

$ local_bash
백그라운드 셸 명령. 서브프로세스로 생성됩니다.
A local_agent
로컬 서브에이전트. 별도의 QueryEngine으로 비동기 실행됩니다.
R remote_agent
원격 워커 에이전트. Bridge/Server를 통해 통신합니다.
T in_process_teammate
Swarm 팀원. 같은 프로세스 내에서 메시지 기반으로 작동합니다.
W local_workflow
스크립트 워크플로우. 사전 정의된 작업 흐름을 실행합니다.
M monitor_mcp
MCP 서버 모니터링. 서버 상태를 감시합니다.

↓ ↓ ↓ (메시지 큐)

W2 Worker 2

Security Model

Claude Code는 여러 층의 보안 메커니즘을 갖추고 있습니다.

Security Layers

1 Permission System
Fail-closed 모델. 모든 쓰기/실행 작업에 명시적 승인 필요. 읽기만 기본 허용.
2 Sandbox Mode
격리된 실행 환경. 파일시스템과 네트워크 접근을 제한합니다.
3 Bash Security
Shell AST 파싱으로 명령어를 분석합니다. 파괴적 명령 감지 및 차단.
4 Policy Enforcement
조직 정책(MDM)을 통한 중앙 집중식 보안 제어.

Dangerous Operations

다음 작업들은 특별한 주의가 필요합니다:
  • 사용자 요청 없이 커밋하지 않음
  • --no-verify로 훅 건너뛰지 않음
  • git 설정을 수정하지 않음
  • 셸 허용 목록을 경솔하게 확장하지 않음
  • 내부 정보를 공개 출력에 노출하지 않음
  • 공유 메모리에 시크릿을 저장하지 않음

Claude Code 소스코드에서 발견되는 주요 아키텍처 패턴들입니다.

Registry-Driven Architecture
Commands, Tools, Skills, Plugins, Hooks가 디렉토리에서 자동 검색되고, lazy load() 함수로 지연 로딩됩니다. 런타임에 필터링, 병합, 소스 기반 중복 제거가 수행됩니다.
Boundary-First Design
Input, Commands, Tools, Services, Utils, UI 각 레이어가 명확히 분리됩니다. 권한 컨텍스트가 전체 쿼리 실행 흐름에 걸쳐 스레딩됩니다.
Context Threading
ToolUseContext가 쿼리 실행 전체에 전달됩니다. AppState, abort controller, 파일 캐시, 권한 규칙을 포함하여 도구가 최신 설정에 접근할 수 있습니다.
Message-Driven Swarm
Unix에서는 UDS 소켓, Windows에서는 메일박스 패턴으로 팀원들이 통신합니다. 샌드박스 권한은 권한 브릿지를 통해 핸드오프됩니다.
File State Caching
FileStateCache (LRU, 설정 가능한 크기)로 반복 읽기를 방지합니다. 메인 스레드와 서브에이전트 간에 공유되며, "merge" 연산을 지원합니다.
Feature Flag System
빌드 타임 플래그(bun:bundle feature())로 미사용 코드를 DCE 제거합니다. 런타임에는 GrowthBook으로 동적 플래그를 관리합니다.
