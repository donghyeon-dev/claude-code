## 1. Claude Code란? (비개발자를 위한 소개)

### AI 코딩 어시스턴트란?

Claude Code는 Anthropic이 만든 AI 코딩 도구입니다. 쉽게 말해, **당신의 아이디어를 코드로 바꿔주는 AI 비서**입니다.

기존에 앱이나 웹사이트를 만들려면 다음이 필요했습니다:
- 수년간의 프로그래밍 공부
- 복잡한 개발 환경 설정
- 프레임워크와 라이브러리 학습
- 디버깅과 테스트 경험

Claude Code를 사용하면 이 과정을 대폭 단축할 수 있습니다. 완전히 대체되는 것은 아니지만, 아이디어를 빠르게 프로토타입으로 만드는 데 매우 강력한 도구입니다.

### 터미널에서 동작하는 대화형 도구

터미널(Terminal)은 컴퓨터에 텍스트 명령을 입력하는 검은 화면입니다. Mac에서는 "터미널" 앱, Windows에서는 "PowerShell" 또는 "명령 프롬프트"가 이에 해당합니다.

Claude Code는 이 터미널에서 실행됩니다. 채팅하듯이 요청을 입력하면 Claude Code가 코드를 작성하고, 파일을 만들고, 심지어 명령을 실행하기도 합니다.

```
예시 대화:

당신: "사용자가 이메일과 비밀번호로 로그인할 수 있는 웹앱을 만들어줘"

Claude Code: (코드를 자동으로 작성하고 파일을 생성합니다)
```

### "말로 설명하면 코드로 만들어주는" 도구

Claude Code의 핵심 가치는 **자연어로 소통**할 수 있다는 것입니다. 전문 용어를 몰라도 됩니다.

- "버튼을 눌렀을 때 팝업이 뜨게 해줘" → 팝업 기능 구현
- "사용자 목록을 테이블로 보여줘" → 테이블 UI 생성
- "이 에러가 뭔지 모르겠어, 고쳐줘" → 에러 분석 및 수정

### 어떤 종류의 프로젝트를 만들 수 있나요?

Claude Code로 만들 수 있는 프로젝트 유형:

**웹 서비스**
- 랜딩 페이지 (회사 소개, 제품 홍보)
- SaaS 웹앱 (할일 관리, 일정 관리, CRM 등)
- 전자상거래 쇼핑몰
- 포트폴리오 사이트

**자동화 도구**
- 데이터 수집 스크립트
- 이메일 자동 발송 시스템
- 엑셀/CSV 파일 처리 자동화
- API 연동 자동화

**앱과 확장 프로그램**
- 크롬 브라우저 확장 프로그램
- 간단한 모바일 앱 (React Native)
- 데스크탑 앱

**백엔드 서비스**
- REST API 서버
- 데이터베이스 설계 및 구축
- 인증 시스템

## 2. 시작하기 전 준비

### 필요한 것들

시작하기 전에 준비해야 할 것들을 단계별로 안내합니다.

**필수 준비물:**
1. 컴퓨터 (Mac 또는 Windows)
2. 인터넷 연결
3. Claude 계정 (claude.ai)
4. Claude Pro 또는 Max 구독 (권장) 또는 API 키

### Claude Code 설치하기

**1단계: Node.js 설치**

Node.js는 Claude Code가 동작하기 위해 필요한 소프트웨어입니다. 마치 앱을 실행하기 위해 운영체제가 필요한 것처럼요.

- https://nodejs.org 에 접속합니다
- "LTS" 버전을 다운로드하고 설치합니다 (LTS = 안정적인 버전)
- 설치 후 터미널을 열고 다음을 입력해서 확인합니다:
  ```
  node --version
  ```
  버전 번호가 나오면 성공입니다.

**2단계: Claude Code 설치**

터미널을 열고 다음 명령어를 입력합니다:

```
npm install -g @anthropic-ai/claude-code
```

이 명령어는 "npm(소프트웨어 저장소)에서 claude-code를 전체(-g) 설치해줘"라는 의미입니다.

**3단계: 설치 확인**

```
claude --version
```

버전 번호가 나오면 설치 성공입니다.

### API 키 설정 또는 Claude Pro/Max 구독

Claude Code를 사용하려면 다음 중 하나가 필요합니다:

**옵션 A: Claude Pro/Max 구독 (초보자에게 권장)**
- claude.ai에서 Pro ($20/월) 또는 Max ($100/월) 구독
- 별도의 API 키 설정 없이 바로 사용 가능
- 월정액이므로 비용 예측이 쉬움
- 처음에는 Pro로 시작해서 필요하면 Max로 업그레이드 권장

**옵션 B: API 키 사용 (사용량에 따라 과금)**
- console.anthropic.com에서 API 키 발급
- 터미널에서 다음 명령어로 설정:
  ```
  export ANTHROPIC_API_KEY="your-api-key-here"
  ```
- 사용한 만큼만 비용이 발생하지만, 비용 모니터링이 필요

### 기본 터미널 사용법

터미널이 처음이라면 이것만 알면 됩니다:

**폴더 이동 (cd = Change Directory)**
```
cd 폴더이름        # 해당 폴더로 이동
cd ..             # 상위 폴더로 이동
cd ~/Desktop      # 바탕화면으로 이동
```

**현재 위치 확인 (ls = List)**
```
ls               # 현재 폴더의 파일/폴더 목록 보기
ls -la           # 숨김 파일까지 자세히 보기
```

**폴더 만들기 (mkdir = Make Directory)**
```
mkdir 내프로젝트   # "내프로젝트"라는 폴더 만들기
```

### 프로젝트 폴더 만들기

모든 프로젝트는 독립된 폴더에서 시작하는 것이 좋습니다.

```
# 바탕화면에 프로젝트 폴더 만들기
cd ~/Desktop
mkdir 내MVP프로젝트
cd 내MVP프로젝트

# 이제 이 폴더에서 Claude Code 시작
claude
```

터미널에 `claude`를 입력하면 Claude Code가 실행되고 대화를 시작할 수 있습니다.

## 3. 아이디어 → MVP 파이프라인

MVP(Minimum Viable Product)는 "최소한의 기능을 갖춘 제품"입니다. 모든 기능을 완벽하게 만들기 전에, 핵심 기능만으로 빠르게 만들어 검증하는 방법입니다.

### 단계 1: 아이디어 정리 (CLAUDE.md에 비전 작성)

프로젝트를 시작하기 전에, 당신의 아이디어를 글로 정리합니다. 프로젝트 폴더에 `CLAUDE.md` 파일을 만들고 다음 내용을 작성합니다:

```markdown
# 프로젝트 비전

## 무엇을 만드나요?
할일을 팀원과 함께 관리할 수 있는 웹앱

## 누가 사용하나요?
- 5~10명 규모의 스타트업 팀
- 프리랜서와 클라이언트

## 핵심 기능 (MVP)
1. 할일 추가/수정/삭제
2. 담당자 지정
3. 마감일 설정
4. 완료 체크

## 나중에 추가할 기능 (MVP 이후)
- 알림 기능
- 파일 첨부
- 댓글 기능

## 기술 선호사항
- 웹 기반 (모바일도 잘 보이면 좋겠음)
- 가능하면 무료로 배포하고 싶음
```

이렇게 정리하면 Claude Code가 프로젝트를 더 잘 이해하고 적절한 기술을 선택합니다.

### 단계 2: /plan 명령어로 기술 계획 수립

Claude Code를 실행한 후 `/plan` 명령어를 사용하면 Claude가 프로젝트에 맞는 기술 계획을 세워줍니다.

```
claude> /plan

(Claude Code가 당신의 프로젝트를 분석하고 질문합니다)

Claude: 어떤 종류의 앱을 만들고 싶으신가요?
당신: 팀 할일 관리 웹앱이요

Claude: 사용자 로그인 기능이 필요한가요?
당신: 네, 이메일로 로그인하고 싶어요

Claude: 데이터는 어디에 저장할까요? (비용이나 설정이 간단한 것을 원하신다면 Supabase를 추천합니다)
당신: 추천해주는 대로 할게요
```

플랜 과정이 끝나면 Claude Code는 어떤 기술을 사용할지, 파일 구조는 어떻게 할지 등을 정리해줍니다.

### 단계 3: autopilot 모드로 자동 구현

계획이 완성되면 autopilot 모드로 실제 코드를 자동으로 만들 수 있습니다:

```
claude> autopilot: 위 계획대로 MVP를 만들어줘. 완성될 때까지 멈추지 말고 진행해줘.
```

Autopilot 모드에서 Claude Code는:
1. 필요한 파일과 폴더를 자동으로 생성합니다
2. 코드를 작성합니다
3. 에러가 있으면 스스로 수정합니다
4. 테스트를 실행합니다
5. 완성될 때까지 계속 진행합니다

이 과정은 프로젝트 규모에 따라 10분~수 시간이 걸릴 수 있습니다.

### 단계 4: 테스트 및 수정

Autopilot이 완료되면 직접 앱을 실행해서 확인합니다:

```
claude> 앱을 실행해서 어떻게 실행하는지 알려줘
```

Claude Code가 실행 방법을 안내해줍니다. 앱을 열어보고 마음에 들지 않는 부분을 대화로 수정합니다:

```
claude> 버튼 색깔을 파란색으로 바꿔줘
claude> 로그인 화면에 로고를 추가해줘
claude> 할일 목록이 날짜 순으로 정렬되게 해줘
```

### 단계 5: 배포

완성된 앱을 인터넷에 올리는 과정입니다:

```
claude> 이 앱을 Vercel에 무료로 배포하는 방법을 단계별로 알려줘
```

Claude Code가 배포 과정을 안내해줍니다. 보통 다음 순서입니다:
1. GitHub에 코드 올리기
2. Vercel(또는 Railway) 계정 만들기
3. GitHub 저장소 연결하기
4. 자동 배포 완료

## 4. 실전 시나리오

### 시나리오 A: SaaS 웹앱 만들기

**목표:** 할일 관리 앱 (Next.js + Supabase)

**Next.js**는 웹 페이지를 만드는 인기 있는 도구이고, **Supabase**는 데이터를 저장하는 데이터베이스 서비스입니다. 둘 다 무료 시작 플랜이 있습니다.

**1단계: 프로젝트 시작**
```
mkdir todo-app
cd todo-app
claude
```

**2단계: 첫 번째 프롬프트**
```
할일 관리 앱을 만들어줘.
- Next.js와 Supabase를 사용해줘
- 사용자가 이메일로 회원가입/로그인할 수 있어야 해
- 할일을 추가, 수정, 삭제할 수 있어야 해
- 할일에 마감일과 우선순위를 설정할 수 있어야 해
- 완료한 할일은 체크하면 목록에서 흐리게 표시해줘
```

**3단계: 설정 도움 요청**
```
Supabase 프로젝트 설정하는 방법을 단계별로 알려줘. 처음이라 아무것도 모름.
```

**4단계: 실행 확인**
```
앱이 제대로 동작하는지 localhost에서 실행해줘. 어떻게 하는지도 알려줘.
```

**5단계: 디자인 개선**
```
전체적인 디자인을 더 깔끔하게 만들어줘. 모던하고 미니멀한 느낌으로.
```

**예상 소요 시간:** 2~4시간 (처음 시도 기준)

**예상 비용:**
- Claude Pro 구독 ($20/월) 사용 시: 추가 비용 없음
- Supabase: 무료 플랜으로 시작 가능
- Vercel 배포: 무료 플랜 가능

---

### 시나리오 B: 랜딩 페이지 만들기

**목표:** 스타트업 랜딩 페이지 (Tailwind CSS + Next.js)

**Tailwind CSS**는 예쁜 디자인을 쉽게 만들 수 있는 도구입니다. **Next.js**와 함께 사용하면 빠르고 모던한 랜딩 페이지를 만들 수 있습니다.

**1단계: 회사 정보 정리**

먼저 `CLAUDE.md`에 회사 정보를 작성합니다:
```markdown
## 회사 정보
- 회사명: 스파크테크
- 제품: AI 기반 고객 지원 챗봇
- 슬로건: "24시간 잠들지 않는 고객 지원"
- 주요 기능: 자동 응답, 다국어 지원, CRM 연동
- 타겟 고객: 중소기업 고객 지원 팀장
- CTA(행동 유도): 무료 체험 신청
```

**2단계: 첫 번째 프롬프트**
```
우리 스타트업 랜딩 페이지를 만들어줘.
- Next.js와 Tailwind CSS 사용
- 반응형 디자인 (모바일, 태블릿, PC 모두 잘 보여야 함)
- 모던하고 세련된 디자인
- 섹션 구성: 히어로(메인 배너), 기능 소개, 가격, 고객 후기, 문의 폼
- 색상 테마는 파란색 계열로
```

**3단계: 내용 채우기**
```
CLAUDE.md에 있는 회사 정보를 바탕으로 랜딩 페이지의 모든 텍스트를 채워줘
```

**4단계: 이미지 및 아이콘 추가**
```
각 기능 섹션에 어울리는 아이콘을 추가해줘. Heroicons나 Lucide 아이콘을 사용해줘.
```

**5단계: 문의 폼 연결**
```
문의 폼을 이메일로 받을 수 있게 해줘. Formspree 서비스를 사용해줘 (무료).
```

**6단계: 배포**
```
이 랜딩 페이지를 Vercel로 배포하는 방법을 알려줘
```

**예상 소요 시간:** 1~3시간

**결과물:** 실제 인터넷에서 접속 가능한 전문적인 랜딩 페이지

### 시나리오 C: API 서비스 만들기

**목표:** 외부 API를 감싸는 REST API 서버 (Express.js)

**API**는 서로 다른 소프트웨어가 통신하는 방법입니다. 예를 들어, 날씨 앱이 날씨 데이터를 가져오는 방법이 API입니다. **Express.js**는 API 서버를 빠르게 만드는 도구입니다.

**예시 시나리오:** 여러 날씨 API를 하나로 묶어서 제공하는 서비스

**1단계: 프롬프트**
```
외부 API를 감싸는 REST API 서버를 만들어줘.
- Express.js 사용
```

## 6. 비용 관리 & 효율적 사용

### Claude Code 비용 구조 이해

Claude Code의 비용은 사용 방식에 따라 다릅니다:

**월정액 구독 방식 (초보자 권장)**
| 구독 | 월 비용 | 특징 |
|------|---------|------|
| Claude Pro | $20 | 일반 사용에 충분 |
| Claude Max (5x) | $100 | 많은 사용량 |
| Claude Max (20x) | $200 | 전문가 수준 |

**API 키 방식 (사용량에 따라 과금)**
- 텍스트 입력/출력 양에 따라 비용 발생
- 초보자에게는 예측이 어려울 수 있음
- 대규모 프로젝트나 팀 사용에 적합

### /cost 명령어로 비용 확인

현재 세션에서 사용한 비용을 확인할 수 있습니다:

```
claude> /cost
```

출력 예시:
```
세션 비용 요약:
- 입력 토큰: 45,230개
- 출력 토큰: 12,450개
- 총 비용: $0.42
```

이를 통해 어떤 작업이 비용이 많이 드는지 파악할 수 있습니다.

### /compact로 컨텍스트 비용 줄이기

대화가 길어지면 Claude Code가 이전 대화를 모두 기억하면서 비용이 증가합니다. `/compact` 명령어를 사용하면 대화 내용을 요약해서 비용을 줄일 수 있습니다:

```
claude> /compact
```

이 명령어는:
1. 현재까지의 대화를 요약합니다
2. 중요한 정보만 유지합니다
3. 컨텍스트 창을 줄여서 이후 비용을 절감합니다

**언제 사용하면 좋을까요?**
- 대화가 1시간 이상 진행된 경우
- 주제가 크게 바뀌는 경우
- 비용이 빠르게 증가하는 것이 느껴질 때

### Agent Tool 사용 시 추가 비용 인지

Claude Code가 "Autopilot" 모드나 복잡한 작업을 수행할 때는 여러 AI 에이전트를 동시에 사용합니다. 이 경우 비용이 더 빠르게 증가할 수 있습니다.

**비용이 많이 드는 작업:**
- Autopilot 모드로 전체 앱 구현
- 대규모 코드 리팩토링
- 여러 파일을 동시에 수정하는 작업

**비용이 적게 드는 작업:**
- 특정 파일 하나만 수정
- 간단한 질문과 답변
- 설명 요청

### 효율적인 세션 관리

**짧고 집중된 세션 권장**

하나의 긴 세션보다 여러 개의 짧은 세션이 더 효율적입니다:

```
# 비효율적 방법
- 하루 종일 하나의 세션에서 모든 것을 처리

# 효율적 방법
- 세션 1 (30분): 기본 구조 만들기
- 세션 2 (30분): 사용자 인증 추가
- 세션 3 (30분): 디자인 개선
```

**세션 시작 시 컨텍스트 제공**

새로운 세션을 시작할 때 이전 진행 상황을 간단히 알려주세요:

```
안녕, 어제 할일 관리 앱을 만들다가 중단했어.
현재 상태: 기본 CRUD 기능은 완성됐고, 사용자 인증이 아직 없어.
오늘은 이메일 로그인 기능을 추가하고 싶어.
```

### Max 요금제 vs API 키 비용 비교

**어떤 경우에 Max 요금제가 유리한가요?**
- 매일 Claude Code를 사용하는 경우
- 대형 프로젝트를 진행하는 경우
- 비용 예측이 중요한 경우
- Autopilot 모드를 자주 사용하는 경우

**어떤 경우에 API 키가 유리한가요?**
- 가끔만 사용하는 경우 (월 몇 시간 이하)
- 팀원과 비용을 나누는 경우
- 정확한 사용량 트래킹이 필요한 경우

**초보자를 위한 추천:**
처음에는 Claude Pro ($20/월)로 시작하세요. 한 달 사용해보고 부족하면 Max로 업그레이드하거나, API 키 방식으로 전환하는 것이 좋습니다.

**주의사항:**
- 중요한 프로젝트라면 반드시 백업 후 시작
- Git을 사용하고 있다면 작업 전 커밋 권장
- 대규모 변경은 작은 단위로 나눠서 진행

---

### "보안은 괜찮나요?"

**Claude Code의 보안 특징:**

**로컬 실행 방식**
- Claude Code는 당신의 컴퓨터에서 직접 실행됩니다
- 코드와 파일은 당신의 컴퓨터에 저장됩니다
- Anthropic의 서버로는 대화 내용만 전송됩니다

**Permission 시스템**
- Claude Code가 파일을 수정하거나 명령을 실행할 때 승인을 요청합니다
- 중요한 작업은 항상 확인하고 승인하세요
- "이 파일을 삭제해도 될까요?"라는 질문이 나오면 신중하게 확인

**주의해야 할 것들:**
- API 키, 비밀번호 등 민감한 정보는 `.env` 파일에만 저장
- `.env` 파일은 절대 GitHub에 올리지 않기 (Claude Code가 자동으로 설정해줌)
- 외부에서 받은 코드는 실행 전에 Claude Code에 "이 코드 안전해?"라고 확인

**실용적인 보안 체크:**
```
이 앱의 기본적인 보안 점검을 해줘. 주요 취약점이 있는지 확인해줘.
```

---

### "Claude Code가 실수를 하면 어떻게 하나요?"

**안심하세요.** Claude Code는 실수를 합니다. 하지만 대부분 수정 가능합니다.

**가장 좋은 안전망: Git**

Git은 코드의 변경 이력을 저장하는 도구입니다. Claude Code와 작업하기 전에 현재 상태를 저장해두면:
- 언제든지 이전 상태로 돌아갈 수 있습니다
- 실험을 두려움 없이 할 수 있습니다

Claude Code에 이렇게 요청하세요:
```
작업 시작 전에 Git으로 현재 상태를 저장해줘. 초보자라 Git을 모르는데 필요한 모든 것을 설정해줘.
```

**실수가 발생했을 때:**
```
방금 수정한 게 동작을 안 해. 이전 상태로 돌려줘.
```
또는
```
Git으로 마지막 커밋 상태로 되돌려줘
```

## 8. 다음 단계

MVP를 만들었다면 더 발전시킬 수 있는 방법들입니다.

### GitHub으로 코드 관리하기

GitHub은 코드를 온라인에 저장하고 관리하는 서비스입니다. 마치 코드를 위한 구글 드라이브입니다.

**왜 GitHub을 사용해야 하나요?**
- 코드 백업 (컴퓨터가 고장나도 안전)
- 변경 이력 추적 (언제 무엇을 바꿨는지)
- 팀원과 협업
- Vercel/Railway와 자동 연동으로 배포 자동화

**시작하기:**
```
GitHub 저장소를 만들고 현재 코드를 업로드하는 방법을 알려줘. GitHub 초보야.
```

### CI/CD로 자동 배포 설정

CI/CD(Continuous Integration/Continuous Deployment)는 코드 변경 시 자동으로 테스트하고 배포하는 시스템입니다.

**실용적인 설명:**
- 코드를 수정하고 GitHub에 올리면
- 자동으로 테스트가 실행되고
- 문제가 없으면 자동으로 배포됩니다

이렇게 하면 수동으로 배포하는 번거로움이 사라집니다.

**설정 요청:**
```
GitHub Actions로 자동 배포 파이프라인을 설정해줘.
코드를 GitHub에 push하면 자동으로 Vercel에 배포되게.
```

### 팀원과 협업하기

팀이 있다면 함께 작업하는 방법을 설정할 수 있습니다:

**GitHub으로 협업:**
- 팀원을 GitHub 저장소에 초대
- 각자 다른 기능을 개발하고 합치기

**Claude Code로 협업 규칙 만들기:**
```
팀 개발을 위한 기본 규칙을 CLAUDE.md에 추가해줘.
코딩 스타일, 커밋 메시지 형식, 브랜치 전략을 포함해서.
```

### 커스텀 스킬로 반복 작업 자동화

자주 하는 작업이 있다면 스킬로 만들어서 한 번에 실행할 수 있습니다.

예를 들어, 새로운 기능을 추가할 때마다 항상 같은 구조를 만들어야 한다면:

```
새 API 엔드포인트를 추가할 때마다 라우터, 컨트롤러, 서비스, 테스트 파일을
자동으로 만들어주는 스크립트를 만들어줘.
사용법: npm run generate:api 이름
```

### 앱을 더 발전시키기

MVP 이후 단계별 발전 방향:

**단계 1: 성능 개선**
```
앱의 로딩 속도를 개선해줘. 현재 어떤 부분이 느린지 분석해줘.
```

**단계 2: 모니터링 추가**
```
앱에 에러 모니터링을 추가해줘. Sentry를 사용하면 좋겠어. 무료 플랜으로 시작하고 싶어.
```

**단계 3: 분석 도구 연동**
```
Google Analytics를 연동해줘. 사용자가 어떤 기능을 많이 사용하는지 알고 싶어.
```

**단계 4: 결제 시스템 추가**
```
유료 구독 기능을 추가하고 싶어. Stripe를 사용하고 싶은데, 초보자도 이해할 수 있게 단계별로 설명하면서 구현해줘.
```
